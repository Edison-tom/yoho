> 面向运营者本人，查用户数、日活、付费、趋势 | 2026-07-04


## 一、数据来源

所有数据已在 Supabase 中。管理后台直接用 `service_role` key 查询数据库，绕过 RLS。

## 二、需要的统计表

### 2.1 补充数据库表（SQL Editor 执行）

```sql
-- 付费记录表
CREATE TABLE IF NOT EXISTS subscriptions (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id) NOT NULL,
    product TEXT NOT NULL,              -- memory_capsule_pro / pet_skin / tree_skin
    plan TEXT,                          -- monthly / yearly / lifetime
    amount NUMERIC NOT NULL,            -- 实付金额（元）
    status TEXT DEFAULT 'active',       -- active / expired / refunded
    started_at TIMESTAMPTZ DEFAULT now(),
    expires_at TIMESTAMPTZ
);

-- 日活记录表（客户端启动时写入一条）
CREATE TABLE IF NOT EXISTS daily_activity (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id) NOT NULL,
    date DATE DEFAULT CURRENT_DATE,
    focus_minutes INT DEFAULT 0,        -- 当日累计专注分钟数
    created_at TIMESTAMPTZ DEFAULT now(),
    UNIQUE(user_id, date)               -- 每人每天一条
);

-- 索引
CREATE INDEX IF NOT EXISTS idx_subs_user ON subscriptions(user_id);
CREATE INDEX IF NOT EXISTS idx_subs_status ON subscriptions(status);
CREATE INDEX IF NOT EXISTS idx_activity_date ON daily_activity(date);
```

### 2.2 客户端改动（一行代码）

Yoho 启动时写入：

```swift
// SupabaseClient.swift 追加
func reportActivity(minutes: Int = 0) async {
    try? await supabase.from("daily_activity")
        .upsert(["user_id": userId, "date": today, "focus_minutes": minutes])
        .execute()
}
```

### SQL 查询（管理员点按钮或定时刷新）：

```sql
-- 注册用户数
SELECT COUNT(*) FROM users;

-- 今日日活
SELECT COUNT(DISTINCT user_id) FROM daily_activity WHERE date = CURRENT_DATE;

-- 付费用户数
SELECT COUNT(DISTINCT user_id) FROM subscriptions WHERE status = 'active';

-- 累计收入
SELECT COALESCE(SUM(amount), 0) FROM subscriptions WHERE status != 'refunded';

-- 30 天日活趋势
SELECT date, COUNT(DISTINCT user_id) as dau
FROM daily_activity
WHERE date >= CURRENT_DATE - INTERVAL '30 days'
GROUP BY date ORDER BY date;

-- 30 天付费趋势
SELECT DATE(started_at) as date, COUNT(*) as count, SUM(amount) as revenue
FROM subscriptions
WHERE started_at >= CURRENT_DATE - INTERVAL '30 days'
GROUP BY DATE(started_at) ORDER BY date;

-- 最近 20 条付费
SELECT s.started_at, u.username, s.product, s.amount
FROM subscriptions s
JOIN users u ON s.user_id = u.id
ORDER BY s.started_at DESC LIMIT 20;
```

## 四、最简单的实现方式

不需要单独部署一个 Web 服务。两个选择：

| 方案 | 怎么做 | 优势 |
|:---|:---|:---|
| **A. Supabase Dashboard 直接查** | SQL Editor 里跑上面的 SQL | 零开发，现在已经能用 |
| **B. 单文件 HTML** | 一个 `admin.html`，内嵌 Supabase JS SDK，用 Chart.js 画图 | 图形化，可分享链接 |
| **C. Yoho App 内置** | 在 macOS App 里加一个隐藏入口，打开管理窗口 | 最方便，随 App 自带 |

推荐 **A 现在就用 + C 后续开发**。SQL 已经写好了，你打开 Supabase SQL Editor 粘贴就能查。

## 五、SQL 视图（一键查询，无需每次写 SQL）

在 Supabase SQL Editor 执行一次，以后点一下就出结果：

```sql
CREATE VIEW IF NOT EXISTS admin_stats AS
SELECT
    (SELECT COUNT(*) FROM users) AS total_users,
    (SELECT COUNT(DISTINCT user_id) FROM daily_activity WHERE date = CURRENT_DATE) AS today_dau,
    (SELECT COUNT(DISTINCT user_id) FROM subscriptions WHERE status = 'active') AS paid_users,
    (SELECT COALESCE(SUM(amount), 0) FROM subscriptions WHERE status != 'refunded') AS total_revenue,
    (SELECT COUNT(*) FROM users WHERE mode = 'single') AS single_users,
    (SELECT COUNT(*) FROM users WHERE mode = 'couple') AS couple_users,
    (SELECT COUNT(*) FROM couples WHERE unbound_at IS NULL) AS active_couples,
    (SELECT COUNT(*) FROM users WHERE mode = 'buddy') AS buddy_users,
    (SELECT COUNT(*) FROM users WHERE mode = 'sis') AS sis_users,
    (SELECT COUNT(*) FROM pairing_codes WHERE status = 'accepted') AS total_pairings,
    (SELECT COUNT(*) FROM teams WHERE disbanded_at IS NULL) AS active_teams;

-- 使用：SELECT * FROM admin_stats;
```

一条 `SELECT * FROM admin_stats` 全部数字出来。

### 5.1 模式分布查询（独立）

```sql
-- 用户模式分布
SELECT mode, COUNT(*) FROM users GROUP BY mode;

-- 活跃情侣对数
SELECT COUNT(*) FROM couples WHERE unbound_at IS NULL;

-- 活跃队伍数（老铁+闺蜜）
SELECT mode, COUNT(*) FROM teams WHERE disbanded_at IS NULL GROUP BY mode;

-- 脱单转化：最近 30 天从 single → couple 的用户数
SELECT COUNT(*) FROM users
WHERE mode = 'couple'
  AND mode_changed_at >= CURRENT_DATE - INTERVAL '30 days';
```
